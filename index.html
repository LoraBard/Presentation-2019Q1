What is JSON?

According to official sources

JSON is a syntax for serializing objects, arrays, numbers, strings, booleans, and null.

Quick facts about JSON syntacsis

  • Data stored in name/value pairs.
  • Records separated by commas. Trailing commas without the following property are not allowed.
  • Double quotes wrap property names and strings. Single quotes are not allowed.

Code Example


							{
								"books": [
								  {
									"id": "bk102",
									"author": "Crockford, Douglas",
									"title": "JavaScript: The Good Parts",
									"genre": "Computer",
									"price": 29.99,
									"publish_date": "2008-05-01",
									"description": "Unearthing the Excellence in JS"
								  }
								]
							  }

Let`s talk about YAML

According to the official documentation

YAML ("YAML Ain't Markup Language") is a human-readable data-serialization language.

Quick facts about YAML syntacsis

  • .yaml files begin with ‘---’, marking the start of the document.
  • key-value pairs are separated by a colon.
  • lists start with a hyphen.
  • YAML uses indentation with one or more spaces to describe nested collections.

Code Example


								books:
								- id: bk102
								  author: Crockford, Douglas
								  title: 'JavaScript: The Good Parts'
								  genre: Computer
								  price: 29.99
								  publish_date: !!str 2008-05-01
								  description: Unearthing the Excellence in JavaScript

Some facts about TOML

TOML is

a data serialisation language designed to be a minimal configuration file format that’s easy to read due to obvious semantics.

Quick facts about TOML syntacsis

  • TOML is case sensitive.
  • A TOML file must contain only UTF-8 encoded Unicode characters.
  • Whitespace means tab or space.

Code Example


									[[books]]
									id = 'bk101'
									author = 'Crockford, Douglas'
									title = 'JavaScript: The Good Parts'
									genre = 'Computer'
									price = 29.99
									publish_date = 2008-05-01T00:00:00+00:00
									description = 'Unearthing the Excellence in JavaScript'

Syntactical differences between TOML, YAML, and JSON

Strings

TOML


							key = "String Value"
							multiline = """\
								The quick brown \
								fox jumps over \
								the lazy dog.\
							"""

YAML


							key : String Value
							multilinePreservedLinebreaks:
							|
								L1 - The quick brown
								L2 - fox jumps over
								L3 - the lazy dog.
							multilineReplaceLinebreaksWithWhitespace:
							>
								This sentence ist just too long to keep it
								on the same line.

JSON


							{
								"key" : "String Value"
							}

Objects / Hash Tables / Collections

TOML


								[table_key]
								property = "Value"
								secondProperty = "2nd Value"

YAML


								objectKey:
								property: Value
								secondProperty: 2nd Value
								alternative: { subkey: 5.0, another: 123 }

Arrays / Lists

TOML


								fruits = [ "Apple", "Banana", "Strawberry" ]
								formats = [
									"YAML",
									"JSON",
									"TOML"
								]

YAML


									fruits:
									- Apple
									- Banana
									- Strawberry
									formats: [ YAML, JSON, TOML ]

JSON


									{
										"fruits": ["Apple","Banana","Strawberry"],
										"formats": [
										"YAML",
										"JSON",
										"TOML"
										]
									}

Conclusion

Anvantages of YAML:

  • YAML has a wide variety of uses and popular for.
  • YAML is the superset of JSON.

Disadvantages of YAML:

  • There are a lot of extra precautions while writing YAML code.
  • JSON and TOML has better performance than YAML.

Anvantages of TOML:

  • TOML was specifically designed to be easier to hand-edit.
  • YAML's syntax is much more complex and subtle than TOML.

Disadvantages of TOML:

  • Not as fast serialization and deserialization as JSON and YAML.
  • TOML can be a little verbose.

Anvantages of JSON:

  • JSON is extra popular.
  • Fast serialization and deserialization.

Disadvantages of JSON:

  • Overly strict.
  • Lack of comments.
  • It doesn’t have any support for multi-line strings.
